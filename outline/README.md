# outline

世界观设定、结构规划与人物卡等元信息放在这里。本目录是写作的“元层控制台”，用来管理规则、结构与方法，而不是放正文。

---

## 文件总览

- `worldview.md` —— 世界观总览：宇宙三公理（A/B/C）、古人与 Reset、暗物质回声等大框架。
- `axioms.md` —— 核心公理：总公理 + 七大部公理 + 章节公理句写作规则与金句仓库。
- `structure-7parts.md` —— 结构骨架：七大部 × 小章提纲，对应后续 `chapters/` 里的章节文件。
- `theme-and-synopsis.md` —— 主题与梗概：母题、叙事定位、读者体验目标、高层故事梗概。
- `core-logic-chain.md` —— 因果主干：从“方便”到“坟墓”的 ABC 逻辑链，以及关键抉择点。
- `characters.md` —— 角色系统：精英线、普通人线、古人回声的功能与人物弧线。
- `writing-notes.md` —— 写作参考：叙事风格、章节七点法、避坑原则与澄清事项。
- `writing-roadmap.md` —— 写作 Roadmap：从序章到第七部的最小可执行任务清单。

---

## 使用建议

- 开局搭结构时：优先看 `worldview.md`、`structure-7parts.md`、`theme-and-synopsis.md`。
- 写具体章节前：先翻 `axioms.md` 找本章公理句，再看 `core-logic-chain.md` 和 `writing-notes.md`。
- 卡在情节选择上：用 `core-logic-chain.md` 检查因果是否自洽，再看 `writing-roadmap.md` 是否被跳步。
- 人物写顺拐时：回到 `characters.md` 对齐角色弧线，避免工具人化。
