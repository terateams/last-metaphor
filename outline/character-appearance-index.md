# 角色出场节点索引 (Character Appearance Index)

> **用途**：写作前的扫描清单。写某个章节时，通过查阅本索引，确定哪些人物必须在场，哪些人物可以作为“相关/陪衬”被调动。

## 一、核心角色出场节点 (Timeline View)

> **阅读指南**：按时间线梳理每部的主视角与核心行动。

### **第一部：出土 (The Excavation)**

- **【林峤】(核心视角)**：亲手挖掘出“不可能层位”的遗物；将地质异常与暗物质信号缝合。
- **【周曼】(技术支持)**：在观测站接收到“第一次回声”；建立早期解码模型。
- **【韩策】(阻碍者)**：代表军方介入，建立封锁线，收缴原始数据。
- **【沈仪】(背景板)**：作为普通市民视角，目睹封锁并感到不安。

### **第二部：归因 (The Attribution)**

- **【林峤】(对抗者)**：解码“10ms 日志”发现 Reset 真相；拒绝修改风险报告。
- **【周曼】(合作者)**：协助林峤验证数据；在解码与合规之间摇摆。
- **【杜衡】(观察者)**：开始介入评估，认为“真相需要被管理”。
- **【许栖】(顾问)**：对遗迹符号进行语义分析，提出“语言污染”警告。

### **第三部：治理 (The Governance)**

- **【杜衡】(主视角)**：发布“文明免疫系统”指标体系；登上权力高点。
- **【林峤】(边缘化)**：警告被无视，转入地下记录；成为系统眼中的“麻烦制造者”。
- **【罗启】(一线)**：初次登场，展示指标体系在基层的执行代价（如强制加班保数据）。
- **【许栖】(隐退)**：意识到公开对抗无效，开始转入地下“黑市”研究。

### **第四部：坍缩 (The Collapse)**

- **【沈仪】(主视角)**：由于“十词语运动”，在家庭和学校中感到严重的“失语症”。
- **【许栖】(地下领袖)**：组织“语言疫苗”播种；与林峤秘密联络。
- **【杜衡】(权威)**：追查地下语料源头，维护指标纯洁性。
- **【苏奕】(新生代)**：登场，作为“适应了模板语”的孩子，与沈仪形成代沟对比。

### **第五部：借贷 (The Debt)**

- **【韩策】(主导者)**：推动军方进行“超因果试航”；导致巨大事故。
- **【周曼】(债务人)**：试图用“回声补丁”修补事故，却导致因果债务激增。
- **【方闻】(激进派)**：登场，作为周曼的激进学生，支持更大胆的实验。
- **【林峤】(记录者)**：冷眼旁观并记录事故真相，积累最终证据。

### **第六部：清算 (The Reset)**

- **【林峤】(高光时刻)**：
  - **听证会**：缺席或仅做简短呈堂。
  - **终局抉择**：在深渊中“哼唱并踩碎发射器”，物理阻断完美优化。
- **【周曼】(关键票)**：在最后关头倒戈，支持切断连接。
- **【杜衡】(被审判)**：在清算中意识到指标体系的荒谬，试图辩护但失败。
- **【苏奕/方闻】**：在混乱中接手一线操作，见证上一代的谢幕。

### **第七部：余生 (The Aftermath)**

- **【苏奕/方闻】(主视角)**：在没有完美指标的废墟上重建生活。
- **【林峤】(符号)**：成为传说中的“守门人”符号；遗物被发掘。
- **【许栖】(遗产)**：她留下的“语言疫苗”成为新文明的文化火种。

## 二、角色出场检查清单 (Checklist)

在开始写一章前，请按以下逻辑扫描：

### 1. 必选检查 (Mandatory)

- **本章的核心冲突是什么？**
  - 涉及“真相/挖掘” -> **林峤** 必须在场或被提及。
  - 涉及“信号/沟通” -> **周曼** 必须提供技术支持。
  - 涉及“规则/评分” -> **杜衡** 必须作为规则化身出现。
  - 涉及“日常/教育/家庭” -> **沈仪** 或 **苏奕** 必须作为受体出现。
  - 涉及“一线/事故/执行” -> **罗启** (前期) 或 **方闻** (后期) 必须在场。

### 2. 相关角色关联 (Relation Chain)

- **如果林峤出场**，且需要冲突 -> 呼叫 **韩策** (高压) 或 **杜衡** (理念冲突)。
- **如果周曼出场**，且需要情感张力 -> 呼叫 **林峤** (老友/理解) 或 **方闻** (师徒/代沟)。
- **如果沈仪出场**，且需要外部压力 -> 呼叫 **杜衡** (指标不仅是数字) 或 **许栖** (救赎的可能性)。

## 三、配角动态填充池 (Dynamic Support Pool)

配角不必每次都具体设定，可根据功能从池中“抓取”并实例化。

### **A. 体制执行者 (The Functionary)**

- **推荐代号**: 官员 A / 审查员 B
- **常用场景**: 听证会、办公室、封锁线
- **行为特征库**:
  - 只有“公事公办”的平板口吻
  - 频繁看表或确认流程
  - 引用规章条款代替回答问题
  - 拒绝眼神接触，只看数据板

### **B. 技术乐观派 (The Technocrat)**

- **推荐代号**: 研究员 C / 实习生 D
- **常用场景**: 实验室、数据中心、发布会后台
- **行为特征库**:
  - 对数据增长表现出单纯的兴奋
  - 对伦理代价毫无敏感度
  - 崇拜杜衡或权威指标
  - 说话夹带大量术语，无法用人话沟通

### **C. 受损的普通人 (The Victim)**

- **推荐代号**: 家长 E / 患者 F / 司机 G
- **常用场景**: 学校门口、医院走廊、故障现场
- **行为特征库**:
  - 语无伦次，逻辑断裂
  - 努力使用“标准语”但表情痛苦
  - 机械重复“系统显示正常”
  - 突然爆发的沉默或崩溃

### **D. 地下异见者 (The Dissident)**

- **推荐代号**: 诗人 H / 卖书人 I
- **常用场景**: 黑市、废弃区、深夜聚会
- **行为特征库**:
  - 使用过时的词汇或修辞
  - 眼神警惕，习惯性观察环境
  - 藏着纸质书或物理存储介质
  - 用隐喻传递信息

**使用原则**：

1. **用完即弃**：除非被判定为“之后还有大用”，否则不给全名，只给职务称呼（如“那个戴眼镜的记录员”）。
2. **功能优先**：先确定他来干什么（阻碍主角？提供线索？展示惨状？），再决定长什么样。
3. **晋升机制**：如果某个配角写得很顺手，可以在 `supporting-cast.md` 中登记转正，赋予固定名字。
