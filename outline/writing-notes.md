# 写作参考

> 用途：作为每次开写前快速扫一眼的“写作防呆卡”。优先做到：**30 秒扫完，知道这一章该干什么、不能干什么。**

---

## 一、速查总则（开写前 30 秒）

### 1.1 我到底在写什么？

- 不是写“AI 有多强”，也不是写“国家有多坏”。
- 写的是这条链条：
  - 社会把“清晰 / 确定性 / 可度量”当成至高美德；
  - 为了清晰，语言被压缩，隐喻变风险，留白变错误，人先失语；
  - 为了更确定，文明不断向因果结构借债；
  - 债务到期，只能通过“重置”来清算因果结构。
- 这就是“人类重置计划”的母题。

> 写作自问：本章的核心冲突，是否都可以还原为“为了更清晰 / 更确定而向未来借债”？

### 1.1.1 荒诞基调开关

- 原则：**越荒诞，越要写得像流程**。
- 荒诞优先通过“通知 / 模板 / 评分 / 口径 / 合规建议”出现，而不是通过作者吐槽出现。
- 最狠的荒诞，是用完全正确、完全专业的语气，说出对人极不体面的事。

> 写作自问：本章最荒诞的一句，是否是以“正常工作语言”说出来的？

### 1.2 三条底层规则（A/B/C）

- **A：语义坍缩**
  - 隐喻 = 风险；留白 = 错误；表达 = 模板。
  - 写法：
    - 让人物主动删减自己的说话方式、情绪、比喻，以符合“可解析模板”。
    - 任何“说不清 / 说复杂”的人，都会在制度面前吃亏。
- **B：指标统治（SAI）**
  - SAI = 语义歧义指数，是主指标；
  - 其余安全 / 稳定 / 幸福 / 增长指标只是外衣、麻醉和燃料。
  - 写法：
    - 每一章想一想：这段里 SAI 是怎么“伸手”的？
    - 让人物因为“SAI 不达标 / 预警 / 被优化”而被迫调整生活。
- **C：因果债务**
  - 每一次“更确定”的技术 / 制度突破，都是在向未来借债；
  - 最后通过“同步性崩坏”（各种系统之间不再对齐）来清算。
  - 写法：
    - 把“当下小优雅”与“未来大代价”绑定在同一个设计决定上。

> 自查：本章有没有至少一处，让读者感到“这个优化以后一定会出大事”，但当下所有人都觉得是进步？

### 1.3 开篇时间点（AI 十年 + 遗迹）

- AI 爆发后约 10 年：
  - 指标从工具 → 变成标准；
  - 标准变制度；
  - 制度吞没日常。
- 遗迹出土：
  - 不是硬塞的宇宙大事，而是“既定走向中的突然阈值”。
  - 让读者相信：没有遗迹，人类也会继续在这条“清晰崇拜”路上走下去，只是走得更慢；有遗迹，等于被命运一脚踹开门。

> 提醒：AI 十年是底色，遗迹是起爆点。写日常先写制度如何渗透，再写遗迹如何打断。

## 二、叙事风格（语气与视角）

1. **基调**

   - 冷静、克制、偏思辨。
   - 情绪在细节中渗出，而不是靠角色大喊大叫。
   - 荒诞感来自“语气太合理”，而不是桥段故意怪异。

2. **展示规则，而非讲解设定**

   - 优先用“具体后果/异常/制度副作用”来呈现世界规则。
   - 少有角色长篇大论解释世界，多用冲突与失败，让读者自然推导出规则。

3. **视角配置**

   - 精英线（科学家/工程师/指标设计官）负责把复杂机制拉清楚。
   - 普通人线（老师/家长/一线工作者/内容创作者）负责把代价写痛。
   - 古人/回声线保持“低带宽、少说话、只给阈值与否定”。

4. **向经典借什么、不借什么**

   - 借：
     - “语言规则 → 叙事规则 → 记忆规则 → 现实规则”的闭环结构；
     - 普通人慢性窒息式进入；
     - 反抗的核心是“保留一块不可结构化的东西”。
   - 不借：
     - 单一邪恶反派、脸谱化极权；
     - 直白影射现实时评。
   - 要写的是：
     - 好意驱动的毁灭；
     - 优化崇拜的坟墓。

## 三、写作方法（章节级操作）

1. **章节七点法（每章检查表）**

   每章最好能回答下面七个问题：

   1. 本章要逼近的**具体问题**是什么？
   2. 本章出现了什么**新证据/新症状**？
   3. 有哪几种**解释在竞争**（科学/政治/伦理/民间）？
   4. 哪个角色做出了**改变局势的行动**？
   5. 本章主要推进 **A/B/C 哪条线**？
   6. 这个行动付出了什么**代价**（资源/关系/道德/债务）？
   7. 结尾给出了什么**新的约束或风险提升**（下一章的钩子）？

2. **章节公理句先行**

   - 每章写前，先写出一句“像宇宙法则/工程原则”的短句。
   - 再用整章剧情去验证或反驳这句话。

   - 公理句格式建议：尽量冷得像合规标语或墙上的制度口号。
     - 例如：
       - “为避免争议，建议减少意义。”
       - “清晰优先，必要时可牺牲现实。”
       - “表达越准确，关系越可控。”
     - 写作时，可以先写一句“如果出现在现实办公楼电梯间，你会以为它是真的标语”的话，再围绕它搭建本章荒诞感。

3. **现实镜像技巧**

   - 可以先用现实中的一个机制/现象作蓝本（绩效考核、家长群、舆论引导等）。
   - 把其中的“可度量指标”拎出来，想象它被抬高 10 倍重要性，会毁掉什么。

4. **每章的硬要求（防止写成设定讲座）**

每一章必须同时出现下面四件事（+1 条荒诞钉子），缺一就是在开设定说明会：

1.  **具体事件**：
    - 有清晰的“发生了什么”：一次会议、一节课、一次抽检、一份被退回的申请等。
2.  **制度介入**：
    - SAI / 口径 / 模板 / 评分系统，如何在这件事中具体伸手？
    - 哪一句话、哪一个表格字段、哪条算法提示改变了走向？
3.  **不可逆代价**：
    - 谁失去了什么？（机会、职位、名誉、关系、自信、记忆方式……）
4.  **线索升级**：

    - 在世界观层面，读者比上一章多知道/多怀疑了一层：
    - 一个更深的约束 / 删节 / 阈值被暗示或明示。

5.  **荒诞钉子**：


    - 至少出现一个“非常合理但非常不对”的细节；
    - 例：家长群发言被系统润色到“情感合规率 98%”；
    - 例：老师收到“隐喻超标预警”；
    - 例：医嘱里禁止使用“差不多”“挺疼”这类词。

> 自查：本章是否出现一个可截图转发的制度荒诞细节？任何只满足 1+2 的片段都优先删减或合并。

## 四、世界观澄清（写作时随时自查）

1. **Reset 的性质**

   - 不是一次性终极毁灭事件，更像反复出现的文明防火墙机制。
   - 目标不是抹除所有痕迹，而是抹除“可复制性”：
     - 留下遗迹与残片作为警告/试纸；
     - 删除关键工程链路与语义框架。

2. **古人的定位**

   - 不是神，也不是全知的文明导师，更像：
     - 被迫沉积在暗物质中的“意识残片 + 约束协议”。
   - 输出规则：
     - 只答关系/阈值/否定；
     - 不给成功范式，不给完全教程。

3. **SAI（语义歧义指数）的角色**

   - 初期是组织内控工具，后来升级为社会级指标。
   - 不是“邪恶技术”，而是**被赋予了错误地位**：
     - 从辅助决策 → 变成决策本身。

4. **AI 与现实的距离**

   - AI 技术描写以“系统行为 + 使用场景”为主：
     - 自动改写/评分作文；
     - 标准化客服话术；
     - 公共政策模拟与推演。
   - 尽量减少需要读者具备高深技术背景才能理解的细节。

## 五、写作避坑（红线提醒）

1. **避免写成时评/论述文**

   - 多用事件、人物、对话承载观点，少用作者旁白和角色演讲。
   - 一旦出现超过两页的“解释段落”，优先拆分为“前后两次失败/冲突”。

2. **避免技术细节自嗨**

   - 不必给出精确的模型参数或算法实现。
   - 重点写：
     - 系统给人的反馈是什么？
     - 政策/制度如何因它而改变？
     - 普通人具体怎么被影响？

3. **避免万能设定解题**

   - 古人回声、暗物质接口、Reset 机制都不能变成“剧情外挂”。
   - 任何“奇迹式解决方案”都要：
     - 带来更大的长期代价，或
     - 仅在非常局部生效，无法复制。

4. **避免脸谱化反派**

   - 反对派、复刻派、清除派、隔离派都应有合理动机：
     - “保护文明”“拯救人类”“为家人好”“维护秩序”等。
   - 让读者在情感上理解他们，而不是简单厌恶。

5. **避免“一切都太合理”**

   - 如果整章读起来像真实政策文件而没有任何刺痛感，说明荒诞感不足；
   - 每一段制度话语里都要藏着一个“人类直觉会反胃”的点，但它在制度语气里完全说得通；
   - 写作时刻意留下一两句“表面体贴、底下凉薄”的话，当作本章的小刀口。

## 六、系统替你说（办公室鬼故事版）

- 经典作品：禁止你说。
- 本书：系统替你说。
  - 不是压制，而是“纠错”；
  - 不是暴力，而是“便利”；
  - 不是恐惧，而是“合规体验更好”。
  - 荒诞不是禁言，而是“服务升级”：
    - 你的话“被照顾了”，所以你不好意思反抗。

写法提示：

- 场景里尽量多写“被自动改写 / 被建议用更标准表达”的瞬间：
  - 家长群 / 内部邮件 / 公开发言都被系统润色成模板句式；
  - 角色在“自己打的原话”和“系统建议版本”之间摇摆，最终屈服于后者。
- 反抗不是勇敢喊出禁词，而是想办法留下一段**无法被模板化的表达**（手写纸条、暗语、隐喻、错误、梦境等）。

可以显式写出“系统常用句式”，当作办公室鬼故事里的固定台词：

- “为避免误解，建议使用如下表述：……”
- “已检测到歧义风险，系统为你优化了表达。”
- “该表述可能引发群体误读，请选择更清晰选项。”
- “你的情绪表达不具可验证性，请参考标准情绪标签。”
- “为提升沟通效率，部分比喻已被替换为直陈句。”
- “根据最新合规口径，你的发言已自动对齐。”

> 每当你写到角色想发一段长话时，先想一想：系统会怎么替 TA 说？TA 又怎样偷偷留下自己的那一点不合规？

## 七、流程化写作 SOP（作者版 SAI 管理）

> 目的：让写作过程本身也被“清晰崇拜”管理一遍，用流程语气把荒诞钉牢。

**步骤一：预检**

- 检查本章是否已设置：
  - 至少一个可量化的“荒诞钉子”（可以被截图、转发、当作笑话讲给同事听）；
  - 一句电梯间级别的“公理标语”。
- 快速翻一眼 `axioms.md` 和 `core-logic-chain.md`，确认今天这段是在推进哪一条 A/B/C 线。

**步骤二：执行**

- 写作中自查：
  - 本章是否有清晰的“具体事件 + 制度介入 + 不可逆代价 + 线索升级 + 荒诞钉子”？
  - 制度介入是否具体到“字段名 / 评分项 / 提示文案”，而不是一句“系统觉得不行”。
  - 有多少段落是“写得很爽但没推进任何 A/B/C 线”？用醒目标记为“爽文段”，待会儿统一处理。

**步骤三：复核**

- 收尾时检查：
  - 本章的“章节公理句”是否被剧情真正支撑，而不是事后贴上去的金句；
  - 代价是否不可逆，且都被角色或制度给出了一套“听上去合理”的解释；
  - 把“爽文段”按照是否服务荒诞 / 代价 / 线索来决定：大改、压缩或删除。

> 使用建议：把本文件当成“写作 debug 面板”，当你觉得故事开始跑偏或情绪过重时，回来对照这些条目，看看是逻辑链、视角、主题还是“荒诞指标”压不住了。

## 八、荒诞语料库（随用随拽）

> 用途：写不出荒诞细节时，从这里随手拎一句，贴进“通知 / 评分 / 润色”场景里，再根据角色和情境做微调。

**1. 通知类（系统给你的善意提醒）**

- “为提升整体幸福感，建议减少隐喻使用频次。”
- “为避免不必要联想，请将表达限制在事实本身。”
- “本次沟通已自动对齐最新合规口径，无需额外解释。”

**2. 评分类（把人类变成指标）**

- “你的情感表述已通过基础合规校验，当前情感清晰度为 92%。”
- “本次互动的关系稳定指数为 0.87，建议保持当前沟通模式。”
- “检测到比喻使用率偏高，已触发‘意义溢出预警’。”

**3. 润色类（删掉你的灵魂但语气更礼貌）**

- “为便于对方理解，你的原句已被简化为标准回复模板。”
- “出于对多方感受的尊重，部分情绪用词已被替换为中性表述。”
- “你的原始输入包含不可验证情绪，已根据上下文生成更稳妥版本。”

> 提醒：这些句子本身也可以作为“荒诞钉子”。写作时尽量让它们以最自然的工作流方式出现，而不是被角色当场吐槽。

## 九、终局立场（写后几部时再看）

- 本书立场：
  - 不反技术，反“单一指标神圣化”；
  - 技术本身是放大器，真正的问题是**只有 SAI 一种“正确”指标**。
- 终局方向：
  - 不是革命胜利，也不是彻底绝望；
  - 而是文明保留足够的“冗余”：
    - 歧义被承认不是缺陷，而是意义的孵化器；
    - 重置被理解为“对过度确定性债务的清算”，而不是随机末日；
    - 文明的重生不靠总计划，而靠一代代人**固执地保留隐喻**。

> 写最后几部 / 终章时的提示：
> 不要去设计一个“完美新文明方案”，而是展示：即便重置之后，仍然有人在偷偷留下难以量化、难以复制的东西。

> 荒诞自检：如果把本章中最荒诞的一句摘出来，读者会不会以为这是现实世界刚发出的某条通知？会，就说明你抓对了气质；不会，就考虑再往“正常工作语气”靠近一点。
