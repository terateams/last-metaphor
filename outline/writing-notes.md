# 写作防呆卡

> **30 秒速查目标**：知道这一章该干什么、不能干什么。  
> **使用时机**：每章开写前的自查清单。  
> **进度管理**：写完某部后见 `writing-roadmap.md`。

---

## 一、核心母题（3 秒锚定）

**不是写"AI 有多强"，也不是写"国家有多坏"。**

写的是这条链：社会把"清晰/确定性/可度量"当至高美德 → 语言被压缩、隐喻变风险 → 文明向因果借债 → 债务到期清算（Reset）。

> **自问**：本章核心冲突能否还原为"为了更清晰/更确定而向未来借债"？

---

## 二、三条底层规则（A/B/C）

| 线              | 核心                                   | 写法要点                                     |
| --------------- | -------------------------------------- | -------------------------------------------- |
| **A：语义坍缩** | 隐喻=风险；留白=错误；表达=模板        | 让人物主动删减说话方式、情绪、比喻以符合模板 |
| **B：指标统治** | SAI 是主指标；安全/幸福/增长是外衣     | SAI 如何"伸手"？衡渊系统是否给出评分/建议？  |
| **C：因果债务** | 每次"更确定"都是借债；清算通过同步崩坏 | 把"当下小优雅"与"未来大代价"绑定在同一设计上 |

> **自查**：本章有没有至少一处让读者感到"这个优化以后一定会出大事"？

## 三、荒诞基调：卡夫卡式荒诞指南

> **核心参照**：卡夫卡的荒诞不是"离奇"，而是"过于正常"。格里高尔变成甲虫后，最荒诞的不是变形本身，而是他醒来第一个念头是"要迟到了"。

### 3.1 卡夫卡式荒诞的六大特征

| 特征 | 卡夫卡原作示例 | 本书应用 |
|------|----------------|----------|
| **官僚迷宫** | 《城堡》中永远到不了的目的地 | SAI评分系统的无限层级、申诉流程的死循环 |
| **罪名不明** | 《审判》中约瑟夫·K不知自己犯了什么罪 | 被标红的表达、被降权的原因永远是"综合评估" |
| **理性的疯狂** | 每个官员都在"正确地"执行职责 | 每个系统提示都在"帮助你"变得更清晰 |
| **身体异化** | 格里高尔无法控制自己的甲壳 | 角色发现自己的语言已经不是自己的语言 |
| **荒诞日常化** | 家人对甲虫的态度从惊恐变成厌烦 | 第一次收到"表达优化建议"时震惊，第100次时习惯 |
| **温柔的暴力** | 《在流放地》中刑具被描述得像精密仪器 | 系统话术永远礼貌、永远"为您考虑" |

### 3.2 三条底层原则（保留并强化）

1. **越荒诞，越要写得像流程**  
   通过"通知/模板/评分/合规建议"出现，而非作者吐槽。  
   → **卡夫卡技法**：《审判》中最恐怖的不是法庭，而是法庭开在出租屋阁楼里，法官在看色情杂志。用"错位的日常感"瓦解荒诞的戏剧性。

2. **最狠的荒诞 = 用完全正确的语气说极不体面的事**  
   例："为提升整体幸福感，建议减少隐喻使用频次。"  
   → **卡夫卡技法**：格里高尔变成甲虫后，叙述者用描述通勤的语气描述他挣扎翻身。语气与内容的"温差"制造窒息感。

3. **系统替你说，而不是禁止你说**  
   不是压制，而是"纠错"；不是暴力，而是"便利"；你的话"被照顾了"，所以你不好意思反抗。  
   → **卡夫卡技法**：《城堡》中没有任何人明确拒绝K，只是"手续还没办完""需要等主管确认""这不归我管"。权力通过缺席运作。

### 3.3 卡夫卡式场景设计四法

**① 目标后退法**
- 角色每走近一步，目标就后退一步，但没有任何人阻拦
- 示例：申诉被受理→转交上级→上级需要补充材料→材料格式已更新→重新提交→申诉被受理……

**② 理由缺席法**
- 最重要的"为什么"永远不被回答，但所有人都表现得理所当然
- 示例："您的内容已被限流。"——"为什么？"——"根据综合评估。"——"什么评估？"——"感谢您的反馈。"

**③ 身份置换法**
- 角色发现自己正在变成自己曾经恐惧/鄙视的人，但这个变化"很合理"
- 示例：沈仪年轻时嘲笑那些紧盯成绩的家长，现在她发现自己每天凌晨三点刷新孩子的SAI评分

**④ 同谋化法**
- 受害者逐渐成为系统的执行者，而且是"自愿的"
- 示例：罗启为了通过合规检查，开始主动举报同事的"表达风险"

### 3.4 避免的伪荒诞

| ❌ 伪荒诞（猎奇式） | ✅ 真荒诞（卡夫卡式） |
|---------------------|------------------------|
| 制度明显邪恶，角色奋起反抗 | 制度看起来合理，角色不知该反抗什么 |
| 官员是脸谱化的坏人 | 官员是尽职的好人，只是"这不归我管" |
| 暴力、血腥、极端惩罚 | 温柔的提示、礼貌的拒绝、无限的等待 |
| "你被禁止说话" | "您的表达已被优化，无需额外操作" |
| 角色愤怒地呐喊 | 角色沉默地点击"我已阅读并同意" |

> **自问**：本章最荒诞的一句是否用"正常工作语言"说出？读者读完会不会说"这很离谱"而是说"这很正常但我说不出哪里不对"？

## 四、每章硬要求（5 件必备）

缺一就是在开设定说明会：

1. **具体事件**：会议/课堂/抽检/被退申请等
2. **制度介入**：SAI/口径/模板如何具体伸手？哪句话/哪个字段/哪条提示改变走向？
3. **不可逆代价**：谁失去什么（机会/职位/名誉/关系/记忆方式）？
4. **线索升级**：读者比上章多知道/多怀疑一层约束或阈值
5. **荒诞钉子（卡夫卡式）**：至少一个"非常合理但非常不对"的细节——  
   - 必须用官方/系统/礼貌语气表达
   - 必须让角色无从反驳（因为每一步都"对"）
   - 必须让读者第一反应是"这好像现实中见过"而非"太夸张了"
   - 示例：不是"你被禁言了"，而是"为保护您的表达质量，本条内容暂不显示"

> **自查**：本章的荒诞钉子，读者转发时会加什么评论？如果是"太离谱了"→失败；如果是"草，这不就是我昨天遇到的"→成功。

## 五、共情系统：不讨好但狠狠刺痛

> **核心原则**：共情 ≠ 讨好；刺痛 ≠ 猎奇；真正的共情是"让读者在自己生活中找到恐惧的锚点"。

### 5.1 三层共情设计

**第一层：日常既视感（"这不就是我吗"）**

- **锚点**：家长群、绩效考核、客服话术、社交媒体发言
- **写法**：
  - 先写读者熟悉的场景（沈仪收到"作文 SAI 不达标"通知）
  - 再加一个"略微扭曲但合理"的细节（系统自动改写后分数提升，但孩子说"这不是我写的"）
  - 最后让角色做出"你也会做"的妥协选择
- **刺痛点**：不是"制度太坏"，而是"我也会屈服"

**第二层：关系断裂（"我说不出爱/怒"）**

- **锚点**：亲子沟通、夫妻争吵、师生对话、朋友倾诉
- **写法**：
  - 设计一个"必须用模板表达情绪"的场景
  - 角色想说的话 vs 系统建议的话，两个版本并列展示
  - 角色选择系统版本后，关系出现不可逆裂痕（但没人能说清为什么）
- **刺痛点**：不是"不能说"，而是"说了但没说"

**第三层：自我消失（"我不记得我是谁"）**

- **锚点**：职业身份、兴趣爱好、童年记忆、梦境
- **写法**：
  - 角色发现自己的记录（日记/邮件/社交账号）被系统"优化"过
  - 试图回忆"原始版本"但失败（因为长期依赖系统表达，已忘记自己怎么说话）
  - 意识到"我的人生是系统替我活的"
- **刺痛点**：不是"被改写"，而是"改写后我认不出自己，但也回不去了"

### 5.2 刺痛的四种节奏

| 节奏       | 触发时机   | 情绪目标 | 写法示例                                                   |
| ---------- | ---------- | -------- | ---------------------------------------------------------- |
| **钝刀**   | 日常积累   | 慢性窒息 | 沈仪每天收到一条"优化建议"，持续一个月                     |
| **尖刀**   | 单次事件   | 突然疼痛 | 罗启在急救现场因"合规话术"延误救人                         |
| **反转刀** | 以为安全时 | 绝望     | 许栖发现自己设计的"富语言疫苗"被系统识别并标红             |
| **长刀**   | 跨章伏笔   | 后怕     | 第一部埋"作文被改写"，第四部揭示"那孩子长大后失去写作能力" |

### 5.3 共情 ≠ 讨好：不给安慰的五条铁律（卡夫卡遗训）

> 卡夫卡从不给读者安慰。《变形记》结尾是家人如释重负地去郊游。《审判》结尾是约瑟夫·K"像一条狗"地死去。

1. **不写"勇敢反抗后胜利"**  
   → 写"反抗后被温柔碾压，最终不得不妥协"  
   → 卡夫卡式追加：角色甚至无法确定自己在反抗什么

2. **不写"制度是邪恶的，我们是无辜的"**  
   → 写"制度是我们一步步选出来的，每一步都很合理"  
   → 卡夫卡式追加：制度的执行者比受害者更无辜，因为他们"只是在工作"

3. **不写"主角团拯救世界"**  
   → 写"主角团通过谎言和背叛，只保住了一小块可能性"  
   → 卡夫卡式追加：主角团可能永远不知道自己保住的那一小块是否有意义

4. **不写"普通人都是受害者"**  
   → 写"普通人既是受害者，也是制度的拥护者"（沈仪家长群困境）  
   → 卡夫卡式追加：普通人对制度的拥护是真诚的，这才是最可怕的

5. **不写"有一个完美答案"**  
   → 写"所有选择都有代价，只能选哪种代价更能活"  
   → 卡夫卡式追加：角色可能到死都不知道自己的选择是对是错

### 5.4 刺痛自查清单

写完一章后问自己：

- [ ] 读者能否在这一章找到"自己可能犯的错"？
- [ ] 角色的妥协是否"如果是我也会这么做"？
- [ ] 制度的荒诞是否"听起来合理但让人反胃"？
- [ ] 代价是否不可逆？（不能靠"后来想明白了"挽回）
- [ ] 是否有至少一处让读者"笑不出来的黑色幽默"？
- [ ] **情绪传达是否靠细节而非形容词？**（见第七节词汇黑白名单）

> **核心检验**：如果读者读完说"好压抑但不敢反驳"，就对了。

## 六、视角配置

- **精英线**（科学家/工程师/指标官）：拉清楚复杂机制
- **普通人线**（老师/家长/一线工作者）：把代价写痛
- **古人回声**：低带宽、只给阈值与否定
- **AI 线（衡渊）**：高频在场但低调，多用报告/评分/提示，少用对白

**人物一致性三自查**：

1. 性别/称谓是否与档案一致？
2. 职业/权限是否匹配判断？
3. 性格动摇是否有铺垫？

## 六·附：普通人场景写作要点

> **功能定位**：普通人线 = 代价放大镜（精英线拉机制，普通人线写痛感）  
> **详细案例库**：见 `notes/ordinary-people-scenes.md`（11 种原型 + 17 个完整场景）

### 核心写作规则（5 条）

1. **先日常，再扭曲，最后妥协**  
   从读者熟悉场景开始 → 加入"略微不对"的制度细节 → 角色做出"你也会做"的选择

2. **用数字/流程语言描述不可量化的事**  
   把情感/关系/记忆用指标重新定义，让读者感到"对但不对"

3. **对比"想说的话"vs"系统建议的话"**  
   两个版本并列展示，让读者看见"灵魂被删减"的过程

4. **用沉默/肢体动作代替语言**  
   当语言被制度污染，肢体成为最后的真实表达

5. **伏笔式刺痛（钝刀慢割）**  
   第一次不强调，多次重复后读者突然意识到"这已经是常态了"

### 四种刺痛节奏

| 节奏         | 触发方式           | 情绪目标   |
| ------------ | ------------------ | ---------- |
| **日常窒息** | 小细节持续累积     | 慢性压迫感 |
| **突然撕裂** | 单次事件打破常态   | 瞬间心碎   |
| **黑色幽默** | 荒诞到笑不出来     | 反胃式共鸣 |
| **后怕伏笔** | 跨章揭示当初的代价 | 追溯式恐惧 |

### 普通人场景自查（8 项）

- [ ] 场景是否从"读者熟悉的日常"开始？
- [ ] 制度介入是否具体到"哪句话被改""哪个字段触发警告"？
- [ ] 角色的妥协是否"如果是我也会这么做"？
- [ ] 是否有"对但不对"的细节（如用数字描述情感）？
- [ ] 读者能否看到"自己的影子"？
- [ ] 情绪冲击来自细节而非作者旁白？
- [ ] 是否有"笑不出来的黑色幽默"或"说不出话的沉默"？
- [ ] **是否用了白名单词汇（动词/名词/感官）而非黑名单（情绪形容词）？**

> **核心检验**：读者说"我也可能这么做，但我不想承认" = 成功。

## 七、写作技巧：词汇黑白名单

> **核心原则**：逼读者靠细节自己推情绪,而不是用形容词告诉读者该有什么感受。

### 7.1 白名单（多用）

**动词**（具体动作,可观察）：

- 抓、缩、抖、撕、摁、按住、憋住、删掉、停顿、撤回、点了"确认"
- 盯着、滑动、刷新、后退、关掉、划掉、圈出、标红、截图
- 咽、顿、皱、扯、捏、松开、低头、侧身、转过去

**名词**（具体物件,可触摸可数）：

- 汗、指节、屏幕边缘、加载圈、评分数字、红条、通知、表格、回执
- 光标、对话框、进度条、勾选项、字段、签名栏、二维码
- 工牌、白大褂、表格、印章、文件夹、U 盘、投影仪

**感官词**（可看见/听见/摸到）：

- 冷风、刺眼的白光、手机震了一下、嗡嗡声、水汽、喉咙干、胃一紧
- 屏幕的蓝光、键盘的咔哒声、空调的气流、走廊的回声、纸张的沙沙声
- 指尖发麻、后颈发凉、太阳穴跳、手心出汗、腿发软

**数字与时间**（可量化的细节）：

- 3 秒、27 条未读、下降 12%、第 5 次、倒数第 3 名、还剩 48 小时
- 0.3 秒延迟、92% 合规率、SAI 68、排名 47/50、已读不回 2 小时

**原则**：能看见、能听见、能摸到、能数出来的词,多用。

### 7.2 黑名单（少用甚至别用）

**抽象情绪词**（直接贴标签式）：

- ❌ 害怕、紧张、愤怒、绝望、羞耻、委屈、悲伤、开心、幸福
- ❌ 焦虑、尴尬、无奈、失落、欣慰、感动、震惊、困惑

**强化副词**（用力过猛的修饰）：

- ❌ 非常、十分、极其、异常、无比、特别、相当、格外、尤其

**空心评判**（作者代替读者下结论）：

- ❌ 很荒诞、很可怕、很压抑、感觉怪怪的、有点难受、整个人都不好了
- ❌ 太离谱了、简直不敢相信、让人毛骨悚然、令人窒息

**原则**：

- 能直接贴在朋友圈上的那种感受句,少用；
- 逼读者靠细节自己推情绪,这才是写作,不是旁白解说。

### 7.3 改写示例（黑名单 → 白名单）

| ❌ 黑名单写法 | ✅ 白名单写法                                              |
| ------------- | ---------------------------------------------------------- |
| 她非常害怕    | 她的手指在屏幕边缘抖了三次,最后还是点了"确认"              |
| 他感到很愤怒  | 他盯着那行字看了 7 秒,删掉重写,又删掉,最后发了个句号       |
| 气氛很压抑    | 会议室里只有空调的嗡嗡声,没人说话,PPT 翻到第 12 页还停在那 |
| 这太荒诞了    | 系统提示："为提升整体幸福感,建议减少隐喻使用频次"          |
| 她非常绝望    | 她看着那个 68 分,又看了一眼"班级均值 94",把手机扣在桌上    |
| 他特别紧张    | 他的喉咙干得像吞了沙子,刷新了第 4 次,评分还是没出来        |

### 7.4 写作自查（每写 500 字查一次）

- [ ] 本段是否出现 3 个以上"黑名单词汇"?（有 → 必须改写）
- [ ] 情绪是否通过动作/感官/数字传达,而非形容词?
- [ ] 是否有"读者能自己推出结论"的细节留白?
- [ ] 删掉所有形容词后,场景是否仍然成立?（不成立 → 依赖形容词,需重写）

> **终极检验**：把所有情绪形容词删掉,如果读者还能感受到情绪,就对了；如果只剩空壳,说明你在用旁白代替写作。

## 八、写作避坑红线

1. **避免时评/论述**：用事件、冲突承载观点,少用作者旁白
2. **避免技术自嗨**：写系统反馈/制度影响,少写算法细节
3. **避免万能设定**：古人回声/Reset 不能是剧情外挂
4. **避免脸谱反派**：所有阵营都有合理动机
5. **避免"一切太合理"**：每段制度话语藏一个"人类直觉反胃"的点
6. **避免情绪旁白**：用白名单词汇写细节,逼读者自己推情绪

### 卡夫卡式特别警示（新增）

7. **避免"恶人作恶"**：卡夫卡笔下没有恶人，只有"照章办事"的普通人。本书也一样——SAI系统没有恶意，官员没有恶意，家长没有恶意，每个人都在"做正确的事"。

8. **避免"问题被解决"**：卡夫卡的故事没有解决方案。约瑟夫·K至死不知道自己的罪名，K永远进不了城堡。本书的角色可以有小胜利，但结构性困境不能被"解决"，只能被"绕过"或"推迟"。

9. **避免"反抗的确定性"**：卡夫卡的主角从不确定自己是在反抗还是在顺从。沈仪不确定自己签字时是在保护孩子还是在出卖孩子；许栖不确定"保留隐喻"是在拯救语言还是在制造噪音。

10. **避免"痛苦的戏剧化"**：卡夫卡式痛苦是钝痛，不是剧痛。格里高尔变成甲虫后最痛苦的不是变形，而是家人渐渐习惯了他的存在。写"痛苦变成日常"，而非"日常变成痛苦"。

## 九、荒诞语料库（随用随拽）

### 8.1 系统善意提醒

- "为提升整体幸福感，建议减少隐喻使用频次"
- "为避免不必要联想，请将表达限制在事实本身"
- "本次沟通已自动对齐最新合规口径，无需额外解释"
- "检测到您的表达可能引发歧义，已为您生成更清晰版本"

### 8.2 把人类变成指标

- "您的情感表述已通过基础合规校验，当前情感清晰度为 92%"
- "检测到比喻使用率偏高，已触发'意义溢出预警'"
- "您与配偶的沟通 SAI 本月下降 12%，建议使用标准情感模板"
- "您孩子的'想象力'指标偏高，可能影响未来合规评估"

### 8.3 删掉灵魂但更礼貌

- "为便于对方理解，你的原句已被简化为标准回复模板"
- "你的原始输入包含不可验证情绪，已根据上下文生成更稳妥版本"
- "检测到您的日记包含 37% 不可验证情绪，已自动转为事实陈述"
- "您的求职信已优化为行业标准格式，通过率提升 68%"

### 8.4 普通人场景专用（新增）

**家长群/教育场景**：

- "您孩子本月作文 SAI 均值 68，建议采用系统优化版本以提升评估表现"
- "班级平均 SAI 达标率 94%，您孩子暂未达标，请关注"
- "为避免影响升学评估，建议减少孩子的'非标准表达'频次"

**职场/绩效场景**：

- "您本季度的沟通清晰度排名倒数 15%，已为您推送标准话术培训"
- "检测到您的邮件包含 3 处情绪化表达，已自动修正"
- "您的客户满意度与话术合规度负相关，建议优先保障合规"

**家庭/关系场景**：

- "检测到您与家人的对话包含 12 处高歧义表达，已生成简化建议"
- "您的情感表达模式与配偶存在 34% 语义偏差，建议对齐"
- "为避免误解，系统已将您的原始表达转换为标准关怀模板"

**医疗/急救场景**：

- "您的急救记录包含 5 处不可验证描述，已按标准流程改写"
- "为保障医疗数据可追溯性，建议使用标准症状描述模板"

### 8.5 焦虑制造者专用话术（新增）

**恐惧营销开场**：

- "震惊！某孩子因 SAI 不达标，被 985 高校拒之门外"
- "最新数据：SAI 低于 75 的孩子，未来就业率暴跌 60%"
- "你的孩子正在被同龄人抛弃，而你还不知道"

**制造紧迫感**：

- "现在不管，孩子将来会恨你一辈子"
- "只剩最后 3 个名额，报满即止"
- "别让孩子输在起跑线，从 SAI 优化开始"

**用模糊数据吓唬人**：

- "根据权威机构调查，SAI 不达标的孩子有 83% 面临……"（不说是哪个机构）
- "教育部最新报告显示……"（实际上只是截取了某个试点区的数据）
- "已有 10 万家长为孩子报名，你还在等什么？"（不说报名的是什么）

**将不确定性变成风险**：

- "虽然政策还没出，但提前准备总没错"
- "万一以后真的影响升学呢？到时候后悔就晚了"
- "别人的孩子都在学，你的孩子不学就是落后"

**打消理性质疑**：

- "有些家长总是质疑，等到孩子吃亏了才知道后悔"
- "你可以不信，但你敢拿孩子的未来赌吗？"
- "现在投资 3999，比将来花几十万补救划算多了"

### 8.6 卡夫卡式场景模板（新增）

> 以下模板直接模仿卡夫卡式荒诞的结构逻辑，可套用于各类场景。

**模板 A：无限后退的目标**
```
角色目标：完成X
第1步：被告知需要先完成Y
第2步：完成Y后被告知Y的格式已更新
第3步：更新格式后被告知需要Z的审批
第4步：Z审批通过后被告知X的申请窗口已关闭
第5步：下一个申请窗口开放时，规则已变
→ 没有任何人说"不"，但X永远无法完成
```

**模板 B：罪名缺席的审判**
```
角色遭遇：被降权/限流/标记/失去某资格
询问原因："根据综合评估"
追问细节："具体指标不便公开"
要求申诉："已记录您的反馈"
申诉结果："经复核，原结论维持"
→ 角色永远不知道自己"错"在哪里，但确实"错"了
```

**模板 C：自愿的同谋**
```
第1阶段：角色鄙视某规则（"这太荒谬了"）
第2阶段：角色被迫遵守（"没办法，大环境如此"）
第3阶段：角色开始理解（"其实有一定道理"）
第4阶段：角色主动执行（"这样对大家都好"）
第5阶段：角色举报他人违规（"我只是照章办事"）
→ 每一步都有"合理"理由，角色成为自己曾经恐惧的人
```

**模板 D：温柔的窒息**
```
系统行为：限制角色的某种能力
系统话术："为了更好地服务您"
角色感受：明知不对，但说不出哪里不对
旁人反应："系统是为你好，你怎么还不满意？"
最终结果：角色开始怀疑是不是自己有问题
→ 暴力被包装成关怀，反抗变成"不识好歹"
```

**模板 E：日常化的异常**
```
第1次：角色震惊（"这怎么可能？"）
第10次：角色习惯（"又来了"）
第100次：角色麻木（不再有反应）
第1000次：角色参与（"新人第一次遇到？正常的"）
→ 荒诞通过重复被"正常化"，最可怕的是连愤怒都消失了
```

## 十、章节公理句设计

**格式要求**：短、冷、可推演，像电梯间的制度标语。

**示例**：

- "为避免争议，建议减少意义"
- "清晰优先，必要时可牺牲现实"
- "表达越准确，关系越可控"
- "凡可量化者，必被武器化"
- "工具不会自己统治你，是你把它变成标准之后，它才开始统治你"

**使用**：每章写前先写公理句，再用剧情验证/反驳它。

## 十一、终局立场（后期提醒）

- **不反技术**，反"单一指标神圣化"
- **不写革命胜利**，写"通过谎言和背叛保住一小块可能性"
- **不写完美新文明**，写"仍有人在偷偷留下难以量化的东西"
- **歧义不是缺陷**，是意义的孵化器
- **文明重生不靠计划**，靠固执地保留隐喻

## 附：卡夫卡式写作速查表

> 写作时随时对照，确保荒诞感的正确调性。

| 卡夫卡原作 | 核心技法 | 本书对应写法 |
|------------|----------|--------------|
| 《审判》约瑟夫·K不知罪名 | 罪名永远缺席 | "您的表达已被限制，原因：综合评估" |
| 《城堡》K永远进不了城堡 | 目标无限后退 | 申诉→补材料→格式更新→重新提交→申诉 |
| 《变形记》格里高尔担心迟到 | 荒诞日常化 | 第100次收到"优化建议"时已不再震惊 |
| 《变形记》家人如释重负 | 受害者被遗忘 | 苏奕被系统"优化"后，没人记得她原来的表达方式 |
| 《在流放地》刑具像精密仪器 | 暴力被美化 | "为提升您的幸福感，已为您优化以下表达" |
| 《城堡》官员永远在工作 | 没有恶人 | 每个SAI审核员都只是"在完成工作" |
| 《饥饿艺术家》无人理解 | 孤独的真相持有者 | 许栖的警告被忽视，因为"数据显示一切正常" |

**一句话自检**：如果你的荒诞场景需要一个"坏人"才能成立，那就不是卡夫卡式荒诞。

## 附：快速自查流程

**开写前（30 秒）**：

1. 翻 `axioms.md` 找本章公理句
2. 确认本章推进 A/B/C 哪条线
3. 预设本章的"卡夫卡钉子"：哪个场景用哪个模板（目标后退/罪名缺席/自愿同谋/温柔窒息/日常化）？

**写作中**：

- 制度介入是否具体到字段名/评分项/提示文案？
- 是否有"写得爽但没推进线"的段落？
- **每 500 字检查一次**：黑名单词汇是否超过 3 个？（超过 → 立即改写）
- **卡夫卡检查**：本段的"权力"是否通过缺席运作？是否有明确的"坏人"？（有 → 考虑改写）

**收尾时**：

- 公理句是否被剧情真正支撑？
- 代价是否不可逆？
- 是否有可截图的荒诞细节？
- 读者能否找到"自己可能犯的错"？（共情自查）
- **词汇黑名单检查**：全文搜索"非常/十分/极其/害怕/紧张/愤怒",有多少个？（超过 3 个 → 必须改写）

> **荒诞自检**：如果把本章最荒诞的一句摘出来，读者会不会以为这是现实世界刚发的通知？会，就对了。

> **卡夫卡自检**：本章有没有任何一个"恶人"？如果有，问自己：把这个人换成一个尽职的普通人，剧情是否仍然成立？如果成立，就换掉"恶人"。

> **共情自检**：读者读完说"好压抑但不敢反驳"？对了。说"好爽主角真棒"？错了，回去重写。

> **词汇自检**：删掉所有情绪形容词,场景是否仍然成立？成立 = 合格；不成立 = 依赖旁白,回去重写。
