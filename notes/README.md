# notes

松散想法、伏笔清单、术语记录、设定推演等放在这里。

## 目录定位

`notes/` 是介于"源代码级 outline"与"公开文本 chapters/docs"之间的**工作区**：

- **不是给读者看的**：可以包含剧透、元讨论、设定推演过程；
- **不是最终设定**：这里的内容可能是草稿、待验证的想法、多个备选方案；
- **是思考结晶**：将对话、头脑风暴、结构性审视转化为可复用的设计笔记。

**与其他目录的关系**：

- `outline/` → 确定的世界观、结构、角色设定（相对稳定）
- `notes/` → 设计过程、伏笔清单、主题分析（动态演化）
- `chapters/` → 读者可见的正文叙事
- `docs/` → 读者可见的世界内文书

## 使用指南

### 何时添加新 note？

当出现以下情况时，考虑创建新笔记：

1. **主题性讨论结晶**：与 Research Agent 或作者自己进行深度头脑风暴后，有系统性洞察需要沉淀
2. **设定推演过程**：某个设定从"想法"到"可写"的推演过程值得记录，供后续参考
3. **伏笔清单**：某条线索需要在多个章节埋伏笔，需要集中管理
4. **场景点子库**：某类场景（如荒诞钉子、普通人困境）的点子积累到一定数量

### 新 note 的命名与格式建议

**命名规则**：Kebab-case，如 `topic-name.md`。

**维护原则**：
1. **定期回顾**：每写完一个部（约 8 章）后，回顾 notes。
2. **去重与整合**：如果多个 notes 涉及同一主题，考虑整合。
3. **状态标记**：在笔记开头标注"草稿 / 待验证 / 已采纳"。
